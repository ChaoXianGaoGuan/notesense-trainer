# 音乐基础训练器功能规格

本文档描述一个音乐基础训练器的通用功能规格。文档不依赖具体编程语言、GUI 框架或当前项目文件结构，可用于在任意工具或平台中重构实现。

## 1. 产品定位

训练器用于帮助学习者建立以下能力：

- 自然音音名与固定唱名的快速对应。
- 以参考音为中心的单音听辨。
- 自然音旋律短句听写。
- 三和弦性质听辨。
- 根音、冠音、音程三者关系的快速计算。

系统包含 5 个训练模块：

```text
看音名选唱名
单音听辨
旋律短句听写
和弦性质听辨
根音冠音音程速算
```

## 2. 看音名选唱名

训练目标：看到自然音音名后，快速选择固定唱名。

固定唱名映射：

```text
C = do
D = re
E = mi
F = fa
G = sol
A = la
B = si
```

题目规则：

- 每题随机显示一个自然音音名：`C D E F G A B`。
- 答案选项为：`do re mi fa sol la si`。
- 避免连续两题完全相同。
- 答对后自动进入下一题。
- 答错后显示正确答案，并等待用户手动进入下一题。
- 用户选择唱名时，可播放该唱名对应音高，建议音区为 C4-B4。

## 3. 单音听辨

训练目标：听目标音高，选择对应音名。用户只需回答音名，不需要回答 octave。

参考音：

```text
C4
```

难度范围：

```text
难度 1：D4 E4 F4 G4 A4 B4 C5
难度 2：D4 E4 F4 G4 A4 B4 C5 D5 E5 F5 G5 A5 B5 C6
难度 3：D4 E4 F4 G4 A4 B4 C5 D5 E5 F5 G5 A5 B5 C6 D6 E6 F6 G6 A6 B6 C7
```

播放方式：

```text
音阶 + Do + 目标
Do + 目标
```

“音阶 + Do + 目标”的推荐播放顺序：

```text
C4 D4 E4 F4 G4 A4 B4 -> C4 -> 目标音
```

错题强化规则：

- 默认关闭。
- 开启后，如果用户误选某个音名，后续优先安排该误选音名对应的合法音高。
- 原题也应进入复习队列。
- 错题队列按难度独立保存。
- 错题强化不得生成当前难度范围外的音高。

## 4. 旋律短句听写

训练目标：听一段短旋律，按顺序输入音名。

音集：

```text
C D E F G A B
```

音区：

```text
C4-B4
```

旋律长度：

```text
2 / 3 / 4 / 5 个音
```

播放方式：

```text
音阶 + Do + 旋律
Do + 旋律
```

交互规则：

- 用户依次输入 `C D E F G A B`。
- 提供 `退格`、`清空`、`提交`。
- 答对后自动进入下一题。
- 答错后显示正确旋律和用户答案，并等待用户手动进入下一题。
- 第一版只训练音高顺序，不训练节奏。

## 5. 和弦性质听辨

训练目标：听三和弦，判断和弦性质。

播放方式：

```text
上行琶音 -> 短暂停顿 -> 齐响和弦
```

和弦性质：

```text
Major 大三       [0, 4, 7]
Minor 小三       [0, 3, 7]
Diminished 减三  [0, 3, 6]
Augmented 增三   [0, 4, 8]
```

阶段设计：

```text
阶段 1：固定 C 根音，只考 Major / Minor
阶段 2：随机自然根音，只考 Major / Minor
阶段 3：固定 C 根音，考 Major / Minor / Diminished / Augmented
阶段 4：随机自然根音，四种性质全考
```

反馈应包含正确性质和组成音，例如：

```text
C Major 大三 = C E G
D Major 大三 = D F# A
```

## 6. 根音冠音音程速算

训练目标：给出根音、冠音、音程三者中的两个，在限时内选择缺失项。

此模块是文本乐理速算，不要求播放音频。

题目形式：

```text
根音：C      冠音：E      音程：？
根音：？      冠音：G      音程：P5 纯五度
根音：D      冠音：？      音程：m3 小三度
```

限时：

```text
5 秒 / 10 秒
默认 10 秒
```

超时规则：

- 超时自动判错。
- 超时后显示正确答案。
- 超时后不允许重复计分。

音名拼写范围：

```text
Cb C C#
Db D D#
Eb E E#
Fb F F#
Gb G G#
Ab A A#
Bb B B#
```

此模块必须区分等音拼写。例如：

```text
C -> E  = M3 大三度
C -> Fb = d4 减四度
```

音程范围：

```text
d2 m2 M2 A2
d3 m3 M3 A3
d4 P4 A4
d5 P5 A5
d6 m6 M6 A6
d7 m7 M7 A7
d8 P8 A8
```

计算原则：

- 音程判断不能只看半音数。
- 必须同时使用字母距离和半音距离。
- `root + interval -> top` 时要保持正确字母拼写。
- `root + top -> interval` 时要根据音名拼写反推音程。
- 生成题目时应验证题目可反推，避免出现无法判定正确答案的组合。

## 7. 音频系统

音频层需要支持以下能力：

```text
播放单音
播放音符序列
播放琶音
播放齐响和弦
```

推荐抽象接口：

```text
play_note(note, octave, duration, volume, timbre)
play_sequence(notes, duration, gap, volume, timbre)
play_chord(notes, duration, volume, timbre)
```

音色建议：

```text
钢琴
尼龙吉他
钢弦吉他
电钢琴
弦乐
长笛
```

全局音频设置：

```text
音量：0-100%
发声时长：80-800 ms
```

不需要音频的模块可以隐藏音频控制。

## 8. 统计规则

每个模块、难度或阶段独立统计：

```text
已答题数
正确数
正确率
当前连续正确数
```

建议统计维度：

```text
看音名选唱名
单音听辨：难度 1 / 2 / 3
旋律短句听写：长度 2 / 3 / 4 / 5
和弦性质听辨：阶段 1 / 2 / 3 / 4
根音冠音音程速算
```

“重新开始”只清空当前模块、当前难度或当前阶段的统计。

## 9. 交互规则

推荐快捷键：

```text
Space = 下一题
R = 重新开始
P = 重听当前题，仅听音、旋律、和弦模块有效
Backspace = 旋律听写退格
```

答案快捷键：

```text
1-7 用于普通七答案模块
```

音程速算模块答案数量较多，可使用鼠标点击、自绘选项面板、搜索式输入或分步选择。若使用数字快捷键，应避免出现不直观的 `0 - =` 之类编号。

## 10. 重构建议

建议将系统拆分为以下层次：

```text
题库与出题逻辑
判题逻辑
统计状态
音频播放
用户界面
```

推荐保留的纯逻辑函数：

```text
note_to_solfege(note)
generate_listen_question(difficulty)
generate_melody_question(length)
build_chord(root, quality)
transpose_note_by_interval(root, interval)
interval_between_notes(root, top)
```

UI 建议：

- 每个训练模块独立渲染自己的设置区和题目区。
- 不建议让所有模块的控件常驻。
- 音频控制只在需要发声的模块显示。
- 音程速算的答案较多，建议使用分组选择、自绘面板或可搜索输入。
- 倒计时应可取消，切换模块或进入下一题时必须清理旧 timer。
