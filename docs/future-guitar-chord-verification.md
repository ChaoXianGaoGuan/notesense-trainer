# 吉他实弹和弦验证计划

本文记录一个未来训练模式的可行方案：屏幕给出目标和弦，用户用吉他实际弹奏，浏览器通过麦克风判断是否弹对。

当前项目尚未实现此功能。本文只作为后续开发参考，不代表现有应用已经接入麦克风输入。

## 功能目标

新增一个输入型训练模块，暂名：

```text
吉他和弦验证
```

模块流程：

1. 页面给出目标三和弦，例如 `D Minor = D F A`。
2. 用户点击开始监听。
3. 用户用吉他弹出该和弦。
4. 浏览器采集 1.5 秒到 2 秒音频。
5. 程序判断实际声音是否匹配目标和弦。
6. 页面反馈正确、接近、错误或无法识别。

推荐定位是“目标和弦验证”，而不是“任意和弦识别”。前者只需要判断用户是否弹对当前题目，后者需要从未知音频中推断和弦名称，难度和误判风险都高很多。

## 浏览器能力

该功能可以继续保持纯前端 PWA，不需要后端上传音频。

可使用的浏览器 API：

- `navigator.mediaDevices.getUserMedia()`：请求麦克风权限并获取音频流。
- Web Audio API：创建 `AudioContext`、`MediaStreamAudioSourceNode` 和 `AnalyserNode`。
- HTTPS 环境：GitHub Pages 是 HTTPS，满足主流浏览器对麦克风权限的要求。

首版应只在本地浏览器内分析音频，不保存、不上传用户录音。

## 推荐算法方向

首版不建议试图识别每根弦、每个 octave 或具体吉他指法，而是识别 12 个 pitch class 的能量分布。

大致数据流：

```text
麦克风输入
  -> 录制短时间窗口
  -> 频谱分析
  -> 汇总为 12 个 pitch class 能量
  -> 与目标和弦音级模板比较
  -> 输出判定结果
```

例如目标 `C Major` 的模板是：

```text
C E G
```

如果检测到的主要音级包含 `C E G`，且额外音能量较低，则判为正确。  
如果检测到 `C G` 但 `E` 很弱，则判为接近。  
如果检测到 `C F A`，则判为错误。

## 可选实现路线

后续实现前可以在三条路线中选择：

1. 原生 Web Audio FFT

   优点是无新增依赖，完全可控。缺点是需要自己写频率到 pitch class 的映射、归一化和阈值逻辑。

2. Meyda

   Meyda 提供浏览器端音频特征提取能力，可用于获取 chroma 等特征。优点是更接近目标算法，缺点是需要新增依赖并验证打包体积。

3. Essentia.js

   Essentia.js 功能更完整，适合更复杂的音频分析。缺点是体积和接入复杂度更高，首版可能偏重。

推荐首版优先评估原生 Web Audio FFT 或 Meyda。只有当首版识别效果明显不足时，再考虑 Essentia.js。

## 首版建议范围

首版只支持：

- Major
- Minor
- Diminished
- Augmented

首版只判断：

- 三个目标音级是否出现。
- 是否有明显额外音级。
- 是否能稳定识别出足够能量。

首版不判断：

- 转位。
- 把位。
- 重复音。
- 具体 octave。
- 每根弦是否正确。
- 节奏、力度或拨弦顺序。

## 反馈设计

建议反馈分为四类：

- 正确：目标三音都被检测到，额外音低于阈值。
- 接近：检测到部分目标音，但缺少一个目标音，或存在轻微额外音。
- 错误：主要音级和目标和弦不匹配。
- 无法识别：输入音量过低、环境噪声过高或频谱不稳定。

接近反馈可以显示：

```text
缺少：E
检测到额外音：F
```

不要在首版承诺过高精度。吉他泛音、房间噪声、琴弦共振、设备麦克风和调音误差都会影响结果。

## 未来实现建议

建议新增独立文件，而不是把麦克风分析写进 React 组件：

```text
src/audio/input-analyzer.ts
src/modules/guitar-chord-verify.ts
```

`src/audio/input-analyzer.ts` 负责：

- 请求麦克风权限。
- 管理 `AudioContext` 和输入流。
- 采集固定时间窗口。
- 输出 pitch class 能量或 chroma 数据。

`src/modules/guitar-chord-verify.ts` 负责：

- 生成目标和弦题目。
- 构造目标 pitch class 模板。
- 根据分析结果判定正确、接近、错误或无法识别。
- 生成统计 key。

React UI 只负责：

- 显示目标题目。
- 触发开始监听。
- 显示监听状态、错误信息和判定反馈。

## 测试建议

未来实现时应优先补单元测试，而不是依赖真实麦克风端到端测试。

建议测试：

- pitch class 映射：不同 octave 的 `C` 都归入同一类。
- 和弦模板：`C Major` 生成 `C E G`。
- 判定逻辑：输入 chroma 含 `C E G` 且额外音低时判正确。
- 缺音逻辑：输入缺少 `E` 时判接近。
- 误弹逻辑：目标 `C E G`，输入 `C F A` 时判错误。
- 无法识别：总能量过低时判无法识别。
- 权限失败：用户拒绝麦克风权限时显示清晰提示。

真实设备验收建议：

- 用常见开放和弦测试 `C`、`D`、`E`、`G`、`A`。
- 测试钢弦吉他和尼龙吉他。
- 测试安静房间和普通环境噪声。
- 测试手机麦克风和电脑麦克风。

## 暂不实现

首版不做：

- 任意自由和弦识别。
- 七和弦、挂留和弦、扩展和弦。
- 判断具体吉他指法。
- 判断节奏、力度、拨弦顺序。
- 后端音频上传。
- 保存用户录音。
